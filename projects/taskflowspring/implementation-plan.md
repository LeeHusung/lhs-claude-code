# TaskFlow Spring — 구현 계획

## 기술 스택

| 영역 | 선택 | 선택 이유 |
|------|------|----------|
| Frontend | React 19 + Vite + TypeScript | Spring Boot API와 분리된 SPA. Vite HMR로 빠른 개발. |
| Backend | Java 25 + Spring Boot 4.0.2 | 고객 지정 스택. Spring Data JPA + Spring Web. |
| Database | H2 (embedded, file 모드) | Spring Boot 네이티브 통합. 설치 불필요. JPA 호환. |
| Styling | Tailwind CSS 4 | 유틸리티 기반 커스텀 디자인 빠른 구현. |
| DnD | @dnd-kit/core + sortable | React 최신 DnD. 접근성 내장, 성능 우수. |
| 차트 | Recharts | React 네이티브 차트. 커스텀 스타일 용이. |

## 디자인 비전

**"Slate & Signal"** — 차분한 슬레이트 배경 위에 선명한 시그널 컬러가 정보 계층을 만드는 정밀한 워크스페이스. 사이드바는 깊은 네이비-차콜(#0f172a), 컨텐츠는 오프화이트(#f8fafc), 주 액센트는 틸(#14b8a6). 상태별 컬러: 앰버(To Do), 블루(In Progress), 바이올렛(Review), 에메랄드(Done). 과도한 그림자/border-radius 지양, 얇은 border 선호. AI 전형 패턴(보라색 그래디언트, 카드 3열, 제네릭 히어로) 명시적 거부.

## 태스크 목록

### Phase A: 프로젝트 스캐폴딩 (순차)

#### Task 0: 프로젝트 초기화 + 디자인 시스템 기반
- 설명: Spring Boot 백엔드 프로젝트 생성 (Java 25, Spring Boot 4.0.2, Spring Web, Spring Data JPA, H2). React + Vite + TypeScript 프론트엔드 프로젝트 생성. Tailwind CSS 설정. "Slate & Signal" 디자인 토큰(색상, 타이포, 간격) 설정. 공통 레이아웃(사이드바 + 헤더 + 메인 영역) 구조 확립. 라우팅 설정. JPA 엔티티 정의 + H2 설정 + 더미 데이터 시드. 프론트-백 CORS 연결 확인.
- 복잡도: 높음
- 의존: 없음
- 참조: spec.md 섹션 2, 2.5, 4.2, 5
- 병렬 가능: 아니오 (모든 후속 태스크의 기반)

### Phase B: 핵심 기능 구현 (병렬 가능)

#### Task 1: 로그인 — 복잡도: 낮음

- 의존: Task 0
- 참조: spec.md 섹션 3.1, 4.1
- 병렬 가능: 예

#### Task 2: 칸반 보드 + DnD — 복잡도: 높음
- 설명: 4개 컬럼(To Do, In Progress, Review, Done) 칸반 보드. @dnd-kit으로 카드 드래그앤드롭. 컬럼 간 이동 시 상태 변경 API 호출. 같은 컬럼 내 순서 변경. 시그널 컬러 적용. 빈 컬럼 표시 처리.
- 의존: Task 0
- 참조: spec.md 섹션 3.2, 4.3
- 병렬 가능: 예

#### Task 3: 업무 CRUD — 복잡도: 중간
- 설명: 업무 생성 폼(모달 또는 패널), 제목/설명/담당자/마감일/우선순위 입력. 유효성 검사. 편집/삭제 기능. 서버 저장 + 영속성.
- 의존: Task 0

#### Task 8: 인앱 알림 + 최종 통합 + 폴리시

- 병렬 가능: 아니오 (최종 통합)

## 의존성 그래프

```
Task 0 (스캐폴딩)
├── Task 1 (로그인)          ─┐
├── Task 2 (칸반 보드+DnD)    ├── Phase B: 병렬
├── Task 3 (업무 CRUD)        ─┘
│
├── Task 4 (상세 패널)        ← Task 2, 3 완료 후
├── Task 5 (팀원 관리)        ─┐
├── Task 6 (필터/검색)        ├── Phase C: 병렬 (Task 7도 여기서 가능)
├── Task 7 (대시보드)         ─┘
│
└── Task 8 (알림+통합+폴리시) ← 최종 순차
```

## 병렬 실행 전략

- **Phase B**: Task 1, 2, 3을 병렬 디스패치 (3개 서브에이전트)
- **Phase C**: Task 4는 B 완료 후 시작. Task 5, 6, 7은 가능한 병렬 (2~3개 서브에이전트)
- **Phase D**: Task 8은 모든 기능 완료 후 순차 실행
- 총 서브에이전트 라운드: 3~4회

## 리스크 및 주의사항

1. **Spring Boot 4.0.2 + Java 25**: 최신 버전이므로 일부 라이브러리 호환성 이슈 가능. Jakarta EE 네임스페이스 사용 필수.
2. **@dnd-kit과 칸반**: DnD 구현이 가장 복잡한 부분. 충분한 시간 할당 필요.
3. **프론트-백 분리**: CORS 설정, API 타입 동기화에 주의.
4. **H2 파일 모드**: 앱 재시작 시 데이터 유지되도록 file 모드 설정 필수. 더미 데이터는 빈 DB일 때만 시드.
5. **데모 일정**: 2026-04-15 (5일 후). 9개 태스크, 병렬 활용 시 충분히 가능한 범위.
