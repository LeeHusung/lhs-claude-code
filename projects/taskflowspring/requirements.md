# 프로젝트 요구사항

## 1. 프로젝트 개요
- **프로젝트명**: TaskFlow
- **고객사**: ABC Corp
- **목적**: 팀 업무 관리 도구. 칸반 보드로 업무를 시각화하고, 팀원 간 업무 할당과 진행 상태를 한눈에 파악할 수 있게 한다.
- **데모 일정**: 2026-04-15

## 2. 핵심 기능 (우선순위순)

1. 로그인: 데모용 간이 로그인 (팀원 선택 방식 또는 이메일/비밀번호). 실제 인증 불필요하지만 "제품답게" 보여야 함 — 우선순위: 필수
2. 칸반 보드: 드래그앤드롭으로 업무 카드를 컬럼 간 이동 (To Do → In Progress → Done 등) — 우선순위: 필수
3. 업무 생성/편집: 제목, 설명, 담당자, 마감일, 우선순위(높음/중간/낮음)를 입력하여 업무 카드 생성 및 수정 — 우선순위: 필수
4. 업무 상세 보기: 카드 클릭 시 모달 또는 사이드 패널로 상세 정보 표시 — 우선순위: 필수
5. 팀원 관리: 팀원 목록 조회, 업무에 담당자 지정 — 우선순위: 권장
6. 필터/검색: 담당자별, 우선순위별, 키워드 검색으로 업무 필터링 — 우선순위: 권장
7. 대시보드: 업무 통계 요약 (상태별 개수, 담당자별 업무량, 마감 임박 업무) — 우선순위: 권장
8. 인앱 알림: 마감 임박, 새 업무 할당 등 앱 내 알림 표시 (이메일 발송은 제외) — 우선순위: 선택

## 3. UI/UX 요구사항
- **디자인 톤**: 모던, 깔끔한 프로페셔널 도구 느낌
- **레퍼런스**: Trello, Linear, Jira 같은 프로젝트 관리 도구 참고
- **반응형**: 불필요 (데스크톱 데모만)
- **핵심 화면**:
  1. 로그인: 팀원 선택 또는 이메일 입력으로 진입
  2. 칸반 보드: 메인 화면. 컬럼별 업무 카드 나열, 드래그앤드롭 지원
  3. 업무 상세: 카드 클릭 시 상세 정보 모달/패널
  4. 대시보드: 업무 현황 통계 차트

## 4. 기술 제약사항
- **지정 스택**: 백엔드는 java 25, spring boot 4.0.2 사용. 나머지는 Planner가 최적 스택 자율 결정.
- **기존 시스템 연동**: 없음
- **배포 환경**: 로컬 데모

## 5. 데모 범위

- **포함**:
  - 데모용 로그인 화면 (팀원 선택 진입)
  - 칸반 보드에서 업무 카드 드래그앤드롭 이동
  - 업무 생성, 편집, 삭제
  - 업무 상세 정보 조회
  - 팀원별 업무 필터링
  - 업무 통계 대시보드
  - 인앱 알림 (마감 임박 등)
- **제외**:
  - 실시간 멀티유저 동기화
  - 파일 첨부
  - 이메일 발송
  - 권한 관리 (어드민/멤버 구분)
- **더미 데이터**: 팀원 5명, 업무 카드 15~20개 (상태별 분산 배치)
