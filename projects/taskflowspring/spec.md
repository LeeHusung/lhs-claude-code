# TaskFlow Spring — 설계 스펙

## 1. 프로젝트 요약

ABC Corp의 팀 업무 관리 도구. 칸반 보드를 중심으로 업무 흐름을 시각화하고, 팀원 간 업무 배분과 진행 상태를 직관적으로 파악할 수 있게 한다. 데스크톱 데모 환경에서 "실제 제품처럼" 보이고 동작하는 것이 핵심 가치. Linear와 Notion의 정밀함을 참고하되 고유한 시각적 정체성을 갖는다.

## 2. 기술 스택

| 영역 | 선택 | 선택 이유 |
|------|------|----------|
| Frontend | React + Vite + TypeScript | Spring Boot API와 분리된 SPA 구조. Vite의 빠른 HMR로 개발 속도 확보. TypeScript로 API 타입 안정성. |
| Backend | Java 25 + Spring Boot 4.0.2 | 고객 지정 스택. Spring Data JPA + Spring Web으로 빠른 REST API 구축. |
| Database | H2 (embedded, file 모드) | Spring Boot와 네이티브 통합. 별도 DB 설치 불필요. 파일 모드로 영속성 보장. JPA 호환. |
| Styling | Tailwind CSS 4 | 유틸리티 기반으로 커스텀 디자인 빠르게 구현. 디자인 시스템 구축에 최적. |
| DnD | @dnd-kit/core + sortable | React 생태계의 가장 현대적인 DnD 라이브러리. 접근성 내장, 성능 우수. |
| 차트 | Recharts | React 네이티브 차트 라이브러리. 커스텀 스타일링 용이. |

## 2.5. 디자인 비전

- **디자인 컨셉**: "Slate & Signal" — 차분한 슬레이트 배경 위에 선명한 시그널 컬러가 정보 계층을 만드는 정밀한 워크스페이스. Linear의 밀도감과 Notion의 여백 균형을 결합한 "프로 도구" 느낌.
- **색상 방향**:
  - 사이드바: 깊은 네이비-차콜 (#0f172a ~ #1e293b)
  - 컨텐츠 영역: 차가운 오프화이트 (#f8fafc)
  - 주 액센트: 틸(Teal, #14b8a6) — 버튼, 링크, 활성 상태에 사용. 보편적인 파란색/보라색을 의도적으로 회피.
  - 상태 컬러: 앰버(To Do), 블루(In Progress), 바이올렛(Review), 에메랄드(Done) — 각 칸반 컬럼의 정체성
  - 우선순위: 로즈(높음), 앰버(중간), 슬레이트(낮음)
  - 텍스트: #0f172a(본문), #64748b(보조), #94a3b8(비활성)
- **타이포그래피**: 시스템 폰트 스택 (Inter가 있으면 우선 적용). 타이트한 자간, 명확한 크기 계층 (14px 본문, 12px 보조, 20px+ 헤딩).
- **절대 사용하지 말 것**:
  - 보라색/파란색 그래디언트 히어로 섹션
  - 동일 크기의 둥근 카드 3열 나열
  - "Welcome to TaskFlow" 류의 제네릭 헤딩
  - 과도한 border-radius (8px 이하 유지, pill 버튼 금지)
  - 그림자 남용 (카드에 무거운 box-shadow 대신 얇은 border 사용)
  - 화려한 장식 일러스트나 아이콘 과다 사용
- **레퍼런스 해석**:
  - **Linear**: 좌측 고정 사이드바 + 우측 슬라이드 패널 구조. 빈틈없는 정보 밀도. 상태 아이콘이 텍스트보다 먼저 눈에 들어오는 스캔 가능한 리스트.
  - **Trello**: 칸반 컬럼의 수평 스크롤. 카드의 간결한 정보 표시 (제목 + 아바타 + 라벨).
  - **Jira**: 필터/검색의 강력함. 대시보드의 통계 밀도.
  - → 이 세 레퍼런스에서 핵심 추출: **사이드바 내비게이션(Linear) + 칸반 수평 레이아웃(Trello) + 데이터 밀도 높은 대시보드(Jira)**

## 3. 기능별 상세 스펙

### 3.1 데모용 로그인 — 우선순위: 필수

**행동 설명**:
앱 진입 시 로그인 화면이 표시된다. 미리 등록된 5명의 팀원 중 한 명을 선택하여 로그인한다. 팀원 선택 시 해당 팀원의 아바타와 이름이 강조되고, "시작하기" 버튼을 누르면 칸반 보드로 이동한다. 세션 동안 로그인 상태가 유지된다 (새로고침 시에도).

**UI 와이어프레임 (텍스트)**:
```
┌──────────────────────────────────────────────┐
│                                              │
│              TaskFlow 로고 + 타이틀            │
│              (중앙 정렬, 미니멀)               │
│                                              │
│    ┌──────┐  ┌──────┐  ┌──────┐             │
│    │아바타 │  │아바타 │  │아바타 │             │
│    │ 이름  │  │ 이름  │  │ 이름  │             │
│    │ 역할  │  │ 역할  │  │ 역할  │             │
│    └──────┘  └──────┘  └──────┘             │
│         ┌──────┐  ┌──────┐                   │
│         │아바타 │  │아바타 │                   │
│         │ 이름  │  │ 이름  │                   │
│         │ 역할  │  │ 역할  │                   │
│         └──────┘  └──────┘                   │
│                                              │
│           [ 시작하기 → ]  (틸 색상)            │
│                                              │
│       배경: 슬레이트 그라데이션 (미묘한)         │
└──────────────────────────────────────────────┘
```

**수락 기준**:
- [ ] 앱 첫 진입 시 로그인 화면이 표시된다
- [ ] 5명의 팀원이 아바타, 이름, 역할과 함께 표시된다
- [ ] 팀원 클릭 시 선택 상태가 시각적으로 구분된다 (테두리 또는 배경 변화)
- [ ] 팀원 선택 없이 "시작하기" 버튼 클릭 시 선택을 유도하는 피드백이 있다
- [ ] 팀원 선택 후 "시작하기" 클릭 시 칸반 보드로 이동한다
- [ ] 로그인 후 사이드바 하단 또는 헤더에 현재 사용자 이름/아바타가 표시된다
- [ ] 새로고침 후에도 로그인 상태가 유지된다 (localStorage 또는 세션)

**엣지 케이스**:
- 이미 로그인된 상태에서 로그인 페이지 접근 시 → 칸반 보드로 리다이렉트
- 로그아웃 기능 → 사이드바에서 로그아웃 클릭 시 로그인 화면으로 복귀

### 3.2 칸반 보드 — 우선순위: 필수

**행동 설명**:
메인 화면. 4개 컬럼(To Do, In Progress, Review, Done)이 수평으로 나열되고, 각 컬럼에 업무 카드가 세로로 쌓인다. 카드를 드래그하여 다른 컬럼으로 이동하면 업무 상태가 변경되고 서버에 저장된다. 같은 컬럼 내에서도 순서를 변경할 수 있다.

**UI 와이어프레임 (텍스트)**:
```
┌────────┬──────────────────────────────────────────────────┐
│        │  칸반 보드              [필터] [검색]   [+ 새 업무] │
│ 사이드  ├──────────┬──────────┬──────────┬──────────┤      │
│ 바     │ ● To Do  │ ● In     │ ● Review │ ● Done   │      │
│        │   (5)    │ Progress │   (2)    │   (4)    │      │
│ ☐ 보드 │          │   (4)    │          │          │      │
│ ☐ 대시 │ ┌──────┐ │ ┌──────┐ │ ┌──────┐ │ ┌──────┐ │      │
│ ☐ 팀원 │ │●높 제목│ │ │●중 제목│ │ │●낮 제목│ │ │✓  제목│ │      │
│        │ │담당 D-3│ │ │담당 D-5│ │ │담당    │ │ │담당    │ │      │
│        │ └──────┘ │ └──────┘ │ └──────┘ │ └──────┘ │      │
│ ──── │ ┌──────┐ │ ┌──────┐ │          │ ┌──────┐ │      │
│ 사용자  │ │●중 제목│ │ │●높 제목│ │          │ │✓  제목│ │      │
│ 아바타  │ │담당 D-7│ │ │담당 D-1│ │          │ │담당    │ │      │
│ 이름   │ └──────┘ │ └──────┘ │          │ └──────┘ │      │
│ 로그아웃│          │          │          │          │      │
└────────┴──────────┴──────────┴──────────┴──────────┘      │
```

**수락 기준**:
- [ ] 4개 컬럼(To Do, In Progress, Review, Done)이 수평으로 표시된다
- [ ] 각 컬럼 헤더에 컬럼명과 카드 개수가 표시된다
- [ ] 각 컬럼에 상태별 시그널 컬러(앰버/블루/바이올렛/에메랄드)가 적용된다
- [ ] 업무 카드에 제목, 우선순위 표시, 담당자 아바타, 마감일이 보인다
- [ ] 카드를 드래그하여 다른 컬럼으로 이동할 수 있다
- [ ] 드래그 시 시각적 피드백이 있다 (카드 반투명, 드롭 영역 하이라이트)
- [ ] 컬럼 간 이동 시 서버에 상태 변경이 저장된다
- [ ] 페이지 새로고침 후에도 변경된 상태가 유지된다
- [ ] 같은 컬럼 내에서 카드 순서 변경이 가능하다
- [ ] 카드 클릭 시 업무 상세 패널이 열린다

**엣지 케이스**:
- 빈 컬럼: "업무가 없습니다" 메시지 또는 빈 드롭 영역 표시
- 카드가 많은 컬럼: 컬럼 내부 스크롤

### 3.3 업무 생성/편집 — 우선순위: 필수

**행동 설명**:
"+ 새 업무" 버튼 클릭 시 업무 생성 폼이 나타난다 (모달 또는 사이드 패널). 제목, 설명, 담당자(드롭다운), 마감일(날짜 선택), 우선순위(높음/중간/낮음) 입력 후 저장하면 칸반 보드의 "To Do" 컬럼에 카드가 추가된다. 기존 카드 편집 시 동일한 폼이 기존 데이터로 채워진 상태로 열린다. 삭제 시 확인 후 카드가 제거된다.

**UI 와이어프레임 (텍스트)**:
```
┌─ 업무 생성/편집 폼 ──────────────────┐
│                              [✕ 닫기]│
│ 제목 *                               │
│ ┌──────────────────────────────┐     │
│ │                              │     │
│ └──────────────────────────────┘     │
│                                      │
│ 설명                                 │
│ ┌──────────────────────────────┐     │
│ │                              │     │
│ │          (여러 줄)            │     │
│ └──────────────────────────────┘     │
│                                      │
│ 담당자          우선순위              │
│ ┌──────────┐   ○ 높음  ○ 중간  ○ 낮음│
│ │ ▾ 선택   │                        │
│ └──────────┘                        │
│                                      │
│ 마감일                               │
│ ┌──────────┐                        │
│ │ 날짜 선택 │                        │
│ └──────────┘                        │
│                                      │
│         [취소]  [저장하기] (틸)        │
└──────────────────────────────────────┘
```

**수락 기준**:
- [ ] "새 업무" 버튼 클릭 시 생성 폼이 나타난다
- [ ] 제목은 필수 입력이며, 빈 제목으로 저장 시 유효성 피드백이 표시된다
- [ ] 담당자 드롭다운에 팀원 5명이 목록으로 나타난다
- [ ] 우선순위를 높음/중간/낮음 중 선택할 수 있다 (기본값: 중간)
- [ ] 마감일을 날짜 선택기로 입력할 수 있다
- [ ] 저장 시 "To Do" 컬럼에 새 카드가 즉시 표시된다
- [ ] 저장된 데이터가 서버에 저장되고 새로고침 후에도 유지된다
- [ ] 기존 카드의 편집 시 기존 데이터가 폼에 채워진 상태로 열린다
- [ ] 편집 저장 시 카드와 상세 정보가 즉시 갱신된다
- [ ] 삭제 시 확인 대화상자가 표시되고, 확인 후 카드가 보드에서 제거된다

**엣지 케이스**:
- 제목 빈 칸으로 저장 시도 → 입력 필드 하단에 에러 메시지
- 아주 긴 제목 → 카드에서 말줄임 처리, 상세에서 전체 표시

### 3.4 업무 상세 보기 — 우선순위: 필수

**행동 설명**:
칸반 보드에서 카드를 클릭하면 우측에서 사이드 패널이 슬라이드하여 열린다 (Linear 스타일). 업무의 모든 정보(제목, 설명, 상태, 우선순위, 담당자, 마감일, 생성일)가 표시된다. 패널 내에서 직접 편집하거나 "편집" 버튼으로 편집 모드 전환이 가능하다. 패널 외부 클릭 또는 닫기 버튼으로 패널을 닫는다.

**UI 와이어프레임 (텍스트)**:
```
┌────────┬─────────────────┬───────────────────┐
│        │  칸반 보드        │ ◀ 업무 상세 패널    │
│ 사이드  │  (반투명 오버레이) │                   │
│ 바     │                  │ 제목 (큰 글씨)      │
│        │                  │ ─────────────      │
│        │                  │ 상태: ● In Progress │
│        │                  │ 우선순위: ●● 높음    │
│        │                  │ 담당자: [아바타] 이름 │
│        │                  │ 마감일: 2026-04-15  │
│        │                  │                   │
│        │                  │ 설명               │
│        │                  │ ───────           │
│        │                  │ 설명 텍스트...      │
│        │                  │                   │
│        │                  │ 생성: 2026-04-10   │
│        │                  │                   │
│        │                  │ [편집] [삭제]       │
└────────┴─────────────────┴───────────────────┘
```

**수락 기준**:
- [ ] 카드 클릭 시 우측에서 패널이 슬라이드 애니메이션으로 열린다
- [ ] 업무의 모든 필드(제목, 설명, 상태, 우선순위, 담당자, 마감일, 생성일)가 표시된다
- [ ] 패널에서 "편집" 버튼 클릭 시 편집 폼으로 전환된다
- [ ] 패널에서 "삭제" 버튼 클릭 시 확인 후 삭제되고 패널이 닫히며 보드가 갱신된다
- [ ] 패널 외부 클릭 또는 ✕ 버튼으로 패널이 닫힌다
- [ ] 상태 배지에 해당 컬럼의 시그널 컬러가 적용된다
- [ ] 마감일이 임박하면 (D-3 이내) 시각적 경고 표시

**엣지 케이스**:
- 설명이 없는 업무 → "설명이 없습니다" 플레이스홀더 표시
- 마감일이 지난 업무 → 마감일이 빨간색으로 표시

### 3.5 팀원 관리 — 우선순위: 권장

**행동 설명**:
사이드바에서 "팀원" 메뉴 클릭 시 팀원 목록 화면으로 이동한다. 각 팀원의 아바타, 이름, 역할, 담당 업무 수가 표시된다. 팀원 카드를 클릭하면 해당 팀원에게 할당된 업무 목록을 볼 수 있다.

**UI 와이어프레임 (텍스트)**:
```
┌────────┬──────────────────────────────────────┐
```

- [ ] 팀원 클릭 시 해당 팀원의 담당 업무 목록이 표시된다

### 3.6 필터/검색 — 우선순위: 권장

**행동 설명**:
칸반 보드 상단에 필터와 검색 컨트롤이 있다. 담당자별 필터(멀티 선택), 우선순위별 필터(멀티 선택), 키워드 검색을 조합하여 카드를 필터링할 수 있다. 필터 적용 시 조건에 맞지 않는 카드가 보드에서 숨겨지고, 활성 필터가 시각적으로 표시된다. 필터 해제 시 전체 카드가 다시 표시된다.

**UI 와이어프레임 (텍스트)**:
```
┌──────────────────────────────────────────────┐
│ 칸반 보드                                     │
│ ┌────────────────────────────────────────┐   │
│ │ 🔍 검색...  │ 담당자 ▾ │ 우선순위 ▾ │ ✕ │   │
│ └────────────────────────────────────────┘   │
│ 활성 필터: [김민수 ✕] [높음 ✕]                  │
│                                              │
│ (필터된 카드만 표시된 칸반 보드)                  │
└──────────────────────────────────────────────┘
```

**수락 기준**:
- [ ] 검색창에 키워드 입력 시 제목/설명에 해당 키워드가 포함된 카드만 표시된다
- [ ] 담당자 필터에서 팀원을 선택하면 해당 팀원의 카드만 표시된다
- [ ] 우선순위 필터에서 선택하면 해당 우선순위 카드만 표시된다
- [ ] 여러 필터를 동시에 적용할 수 있다 (AND 조건)
- [ ] 활성 필터가 칩/태그 형태로 표시되고, 개별 해제할 수 있다
- [ ] 모든 필터 해제 시 전체 카드가 다시 표시된다
- [ ] 필터 결과가 0건일 때 "조건에 맞는 업무가 없습니다" 메시지 표시

**엣지 케이스**:
- 빈 검색어 → 필터 해제와 동일
- 특수문자 검색 → 에러 없이 결과 없음 표시

### 3.8 인앱 알림 — 우선순위: 선택

## 5. 데이터 모델

### 엔티티 목록

| 엔티티 | 핵심 필드 | 설명 |
|--------|----------|------|
| Member (팀원) | id, name, email, role, avatarUrl | 데모용 고정 팀원 5명. 로그인 주체. |
| Task (업무) | id, title, description, status, priority, assignee, dueDate, position, timestamps | 칸반 카드의 실체. 상태 변경과 순서 관리. |
| Notification (알림) | id, recipient, message, type, read, relatedTask, timestamp | 인앱 알림. 마감 임박, 새 할당 등. |

### 엔티티 관계
- Member 1:N Task — 한 팀원이 여러 업무를 담당
- Member 1:N Notification — 한 팀원이 여러 알림을 수신
- Task 1:N Notification — 하나의 업무가 여러 알림을 발생시킴

### 더미 데이터

- **팀원 5명**: 김민수(PM), 이지은(백엔드 개발), 박서준(UI 디자이너), 최예린(프론트엔드 개발), 정하은(QA)
- **업무 18개**: 상태별 분산 — To Do 5, In Progress 4, Review 3, Done 6
- **우선순위 분산**: 높음 5, 중간 8, 낮음 5
- **마감일**: 과거~미래 분산, 3건 이상 D-3 이내
- **알림**: 마감 임박 3건 + 새 할당 2건

## 6. 데모 시나리오

1. **로그인**: 로그인 화면에서 "김민수" 선택 → "시작하기" 클릭 → 칸반 보드 진입
2. **칸반 보드 조작**: "To Do"의 카드를 "In Progress"로 드래그 → 상태 변경 확인 → 새로고침으로 영속성 확인
3. **업무 생성**: "+ 새 업무" 클릭 → 제목/설명/담당자/우선순위/마감일 입력 → 저장 → "To Do"에 카드 표시
4. **업무 상세**: 카드 클릭 → 사이드 패널에서 상세 확인 → 편집으로 제목 변경 → 보드에 반영 확인
5. **필터/검색**: 담당자 "이지은" 필터 → 이지은 업무만 표시 → 우선순위 "높음" 추가 필터 → 조합 필터 확인 → 필터 해제
6. **대시보드**: 사이드바 "대시보드" 클릭 → 상태별 도넛 차트 + 담당자별 바 차트 + 마감 임박 목록 확인
7. **팀원 관리**: 사이드바 "팀원" 클릭 → 팀원 목록 → 팀원 클릭하여 담당 업무 조회
8. **알림**: 알림 벨 클릭 → 마감 임박 알림 확인 → 알림 클릭으로 해당 업무 상세 이동
9. **마무리**: 칸반 보드로 돌아와 전체 보드 상태 확인
