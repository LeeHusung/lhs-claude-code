# TaskFlow — 설계 스펙

## 1. 프로젝트 요약

TaskFlow는 ABC Corp 팀을 위한 업무 관리 도구이다. 칸반 보드를 중심으로 팀의 업무 흐름을 시각화하고, 업무 생성·할당·추적을 한 곳에서 수행한다. 프로젝트 매니저와 팀원 모두가 현재 업무 현황을 한눈에 파악할 수 있으며, 드래그앤드롭으로 직관적으로 상태를 변경할 수 있다. 데모 대상은 ABC Corp 의사결정자이며, "우리 팀이 바로 쓸 수 있겠다"는 인상을 주는 것이 핵심 가치이다.

## 2. 기술 스택

| 영역 | 선택 | 선택 이유 |
|------|------|----------|
| Frontend | Next.js (App Router) | 풀스택 단일 프레임워크로 별도 서버 셋업 불필요. SSR로 빠른 초기 렌더링. 데모 최단 구현에 최적. |
| Backend | Next.js API Routes | 프론트와 동일 프레임워크 내에서 API 구현. 배포·개발 환경 단일화. |
| Database | SQLite | 사용자-업무 관계형 데이터에 적합. 제로 설정, 로컬 데모에 최적. 파일 하나로 데이터 영속성 보장. |
| Styling | Tailwind CSS | 디자인 시스템 빠른 구축. 유틸리티 기반으로 일관된 간격·색상 유지 용이. |

## 2.5. 디자인 비전

- **디자인 컨셉**: "Focused Workspace" — 어두운 잉크빛 캔버스 위에 따뜻한 앰버 액센트가 행동을 유도하는 프리미엄 도구. Linear의 절제된 우아함과 Trello의 직관적 칸반을 결합하되, 독자적인 따뜻한 색감으로 차별화한다.
- **색상 방향**:
  - 캔버스: 푸른 기운이 도는 깊은 잉크색 (blue-tinted black)
  - 표면: 부드러운 차콜 — 카드와 패널은 캔버스에서 미묘하게 떠오르는 느낌
  - 주요 액센트: 따뜻한 앰버/골드 — 버튼, 선택 상태, 중요 하이라이트에 사용
  - 상태 색상: 틸 그린(완료), 스카이 블루(진행 중), 코랄 로즈(높은 우선순위)
  - 텍스트: 오프화이트(본문), 뮤트 그레이(보조 텍스트)
- **절대 사용하지 말 것**:
  - 보라색/파란색 그래디언트 히어로 섹션
  - 동일 크기 둥근 카드 3열 나열 (대시보드 통계도 비대칭 레이아웃 사용)
  - "Welcome to TaskFlow" 같은 제네릭 헤딩
  - 밝은 흰색 배경에 옅은 회색 테두리의 "SaaS 템플릿" 느낌
  - 무지개색 상태 배지
  - stock 느낌의 일러스트/아이콘 남용
- **레퍼런스 해석**:
  - **Linear에서 가져올 것**: 다크 테마의 절제된 우아함, 부드러운 전환 애니메이션, 오른쪽 슬라이드 패널 상세 뷰, 키보드 단축키 느낌의 프로페셔널함
  - **Trello에서 가져올 것**: 칸반 보드의 직관적 구조, 컬럼별 명확한 시각 구분, 드래그앤드롭의 자연스러운 피드백
  - **Jira에서 가져올 것**: 사이드바 네비게이션 구조, 필터/검색의 기능적 배치, 대시보드의 정보 밀도

## 3. 기능별 상세 스펙

### 3.1 로그인 — 우선순위: 필수

**행동 설명**:
앱에 처음 진입하면 팀 워크스페이스 진입 화면이 나타난다. 미리 등록된 5명의 팀원 아바타(이니셜 + 이름)가 표시되고, 사용자가 자신의 아바타를 클릭하면 즉시 칸반 보드로 진입한다. 하단에 "이메일로 로그인" 링크가 있어, 클릭하면 이메일/비밀번호 입력 폼이 나타난다 (데모용 — 아무 값이나 입력해도 진입 가능). 로그인 후 헤더에 사용자 이름과 아바타가 표시된다.

**UI 와이어프레임 (텍스트)**:
```
┌─────────────────────────────────────────────┐
│                                             │
│              [TaskFlow 로고]                 │
│         "팀 워크스페이스에 접속하기"           │
│                                             │
│   ┌──────┐  ┌──────┐  ┌──────┐              │
│   │ 아바타 │  │ 아바타 │  │ 아바타 │           │
│   │ 김민수 │  │ 이지은 │  │ 박준호 │           │
│   └──────┘  └──────┘  └──────┘              │
│        ┌──────┐  ┌──────┐                   │
│        │ 아바타 │  │ 아바타 │                  │
│        │ 최서연 │  │ 정태웅 │                  │
│        └──────┘  └──────┘                   │
│                                             │
│          — 또는 이메일로 로그인 —              │
│                                             │
└─────────────────────────────────────────────┘
```

**수락 기준**:
- [ ] 5명의 팀원 아바타가 이름과 함께 표시된다
- [ ] 아바타 클릭 시 해당 사용자로 로그인되어 칸반 보드 화면으로 이동한다
- [ ] 로그인 후 헤더 영역에 로그인한 사용자의 이름과 아바타가 표시된다
- [ ] 새로고침 후에도 로그인 상태가 유지된다
- [ ] "이메일로 로그인" 전환 시 이메일/비밀번호 폼이 나타나고, 입력 후 로그인 가능하다
- [ ] 로그아웃 기능이 있어 다른 사용자로 전환할 수 있다

**엣지 케이스**:
- 이메일 로그인 시 빈 필드 제출: 입력 요구 안내 표시
- 이미 로그인된 상태에서 로그인 페이지 접근: 보드로 리다이렉트

### 3.2 칸반 보드 — 우선순위: 필수

**행동 설명**:
로그인 후 메인 화면. 가로로 나열된 컬럼들(To Do, In Progress, In Review, Done)에 업무 카드가 배치되어 있다. 각 컬럼 헤더에는 컬럼명과 업무 개수 뱃지가 표시된다. 사용자는 카드를 드래그하여 다른 컬럼으로 이동할 수 있으며, 이동 즉시 상태가 변경되고 서버에 저장된다. 같은 컬럼 내에서도 순서 변경이 가능하다. 각 카드에는 제목, 우선순위 표시(색상 바 또는 뱃지), 담당자 아바타, 마감일이 간략히 표시된다.

**UI 와이어프레임 (텍스트)**:
```
┌──────────────────────────────────────────────────────────┐
│ [사이드바]  │  To Do (5)    │ In Progress (3) │ In Review (2) │ Done (7)     │
│             │ ┌───────────┐ │ ┌───────────┐   │ ┌───────────┐ │ ┌───────────┐│
│ ◉ Board     │ │ 카드 제목   │ │ │ 카드 제목   │  │ │ 카드 제목   │ │ │ 카드 제목   ││
│ ◎ Dashboard │ │ 🟡중간 @김민수│ │ │ 🔴높음 @이지은│  │ │ 🟡중간 @박준호│ │ │ ✅완료 @최서연││
│ ◎ Team      │ │ 4/15 마감   │ │ │ 4/12 마감   │  │ │ 4/14 마감   │ │ │            ││
│             │ └───────────┘ │ └───────────┘   │ └───────────┘ │ └───────────┘│
│ ─────────── │ ┌───────────┐ │                 │               │              │
│ [사용자 정보]│ │ + 업무 추가 │ │                 │               │              │
│             │ └───────────┘ │                 │               │              │
└──────────────────────────────────────────────────────────┘
```

**수락 기준**:
- [ ] 4개 컬럼(To Do, In Progress, In Review, Done)이 가로로 나열된다
- [ ] 각 컬럼 헤더에 컬럼명과 해당 컬럼의 업무 개수가 표시된다
- [ ] 업무 카드를 드래그하여 다른 컬럼으로 이동할 수 있다
- [ ] 드래그앤드롭 시 시각적 피드백이 있다 (드래그 중인 카드 강조, 드롭 위치 표시)
- [ ] 컬럼 이동 후 페이지 새로고침 시에도 변경된 상태가 유지된다
- [ ] 같은 컬럼 내에서 카드 순서 변경이 가능하다
- [ ] 카드에 제목, 우선순위 표시, 담당자 아바타, 마감일이 표시된다
- [ ] 마감일이 임박한 카드(2일 이내)는 시각적으로 구별된다

**엣지 케이스**:
- 빈 컬럼: "업무가 없습니다" 빈 상태 메시지 또는 드롭 영역 표시
- 컬럼에 카드가 많을 때: 컬럼 내 스크롤 지원

### 3.3 업무 생성/편집 — 우선순위: 필수

**행동 설명**:
컬럼 상단 또는 보드 영역의 "업무 추가" 버튼을 클릭하면 업무 생성 폼이 나타난다. 제목(필수), 설명, 담당자(팀원 드롭다운), 마감일(날짜 선택), 우선순위(높음/중간/낮음)를 입력하고 저장한다. 저장 후 해당 컬럼(기본 To Do)에 새 카드가 즉시 나타난다. 기존 카드의 제목이나 속성을 수정할 수 있으며, 삭제도 가능하다. 삭제 시 확인 대화상자가 표시된다.

**UI 와이어프레임 (텍스트)**:
```
┌─────────────────────────────┐
│ 새 업무 만들기           [X] │
├─────────────────────────────┤
│ 제목: [________________]    │
│                             │
│ 설명:                       │
│ [________________________]  │
│ [________________________]  │
│                             │
│ 담당자: [▼ 팀원 선택]       │
│ 마감일: [📅 날짜 선택]      │
│ 우선순위: ○높음 ●중간 ○낮음  │
│                             │
│        [취소]  [저장]       │
└─────────────────────────────┘
```

**수락 기준**:
- [ ] "업무 추가" 버튼 클릭 시 생성 폼이 나타난다
- [ ] 제목, 설명, 담당자, 마감일, 우선순위를 입력할 수 있다
- [ ] 저장 시 새 카드가 To Do 컬럼에 즉시 추가된다
- [ ] 저장된 업무가 새로고침 후에도 유지된다
- [ ] 기존 업무의 각 필드를 수정할 수 있다
- [ ] 업무 삭제 시 확인 대화상자가 나타나고, 확인 후 삭제된다
- [ ] 삭제 후 보드에서 카드가 즉시 사라진다

**엣지 케이스**:
- 제목 빈 상태에서 저장 시도: 제목 필수 입력 안내
- 매우 긴 제목: 카드에서 말줄임(...) 처리, 상세 뷰에서 전체 표시

### 3.4 업무 상세 보기 — 우선순위: 필수

**행동 설명**:
칸반 보드에서 카드를 클릭하면 화면 오른쪽에서 슬라이드 패널이 열리며 업무의 전체 정보가 표시된다. 제목, 설명, 상태(현재 컬럼), 담당자, 마감일, 우선순위가 모두 보이고, 패널 내에서 직접 편집이 가능하다. 패널 닫기(X 버튼 또는 패널 바깥 클릭)로 보드 화면으로 복귀한다.

**UI 와이어프레임 (텍스트)**:
```
┌───────────────────────┬──────────────────────┐
│                       │  업무 상세         [X] │
│   (칸반 보드가         │ ─────────────────── │
│    어둡게 처리됨)      │  제목: API 연동 작업  │
│                       │  상태: In Progress    │
│                       │                      │
│                       │  설명:               │
│                       │  REST API 연동 및     │
│                       │  에러 핸들링 구현...   │
│                       │                      │
│                       │  담당자: 이지은       │
│                       │  마감일: 2026-04-12   │
│                       │  우선순위: 🔴 높음    │
│                       │                      │
│                       │  [편집] [삭제]       │
└───────────────────────┴──────────────────────┘
```

**수락 기준**:
- [ ] 카드 클릭 시 오른쪽에서 슬라이드 패널이 애니메이션과 함께 열린다
- [ ] 패널에 업무의 모든 정보(제목, 설명, 상태, 담당자, 마감일, 우선순위)가 표시된다
- [ ] 패널 내에서 업무 정보를 직접 편집할 수 있다
- [ ] 편집 후 저장하면 칸반 보드의 카드에도 변경이 즉시 반영된다
- [ ] X 버튼 또는 패널 바깥 클릭으로 패널이 닫힌다
- [ ] 패널이 열린 상태에서 배경(보드)은 어둡게 처리되어 포커스가 패널에 있음을 나타낸다

**엣지 케이스**:
- 설명이 매우 긴 경우: 패널 내 스크롤
- 패널 열린 상태에서 브라우저 뒤로가기: 패널 닫힘 처리

### 3.5 팀원 관리 — 우선순위: 권장

**행동 설명**:
사이드바에서 "Team" 메뉴를 클릭하면 팀원 목록 화면으로 이동한다. 5명의 팀원이 카드 형태로 표시되며, 각 카드에는 아바타, 이름, 역할, 현재 담당 업무 수가 나타난다. 팀원 카드를 클릭하면 해당 팀원이 담당하는 업무 목록이 필터링되어 표시된다 (보드 뷰로 전환하며 해당 팀원 필터 적용).

**UI 와이어프레임 (텍스트)**:
```
┌──────────────────────────────────────────────┐
│ [사이드바]  │  팀원 관리                        │
│             │                                  │
│ ◎ Board     │  ┌──────────────────────────┐    │
│ ◎ Dashboard │  │ [아바타] 김민수 — 개발자    │    │
│ ◉ Team      │  │ 담당 업무: 4건 | 진행 중: 2 │    │
│             │  └──────────────────────────┘    │
│             │  ┌──────────────────────────┐    │
│             │  │ [아바타] 이지은 — 디자이너  │    │
│             │  │ 담당 업무: 3건 | 진행 중: 1 │    │
│             │  └──────────────────────────┘    │
│             │  ...                             │
└──────────────────────────────────────────────┘
```

**수락 기준**:
- [ ] 사이드바 "Team" 클릭 시 팀원 목록 화면으로 이동한다
- [ ] 5명의 팀원이 아바타, 이름, 역할과 함께 표시된다
- [ ] 각 팀원의 현재 담당 업무 수가 표시된다
- [ ] 팀원 클릭 시 해당 팀원의 업무만 필터링된 칸반 보드로 이동한다

**엣지 케이스**:
- 담당 업무가 0건인 팀원: "배정된 업무 없음" 표시

### 3.6 필터/검색 — 우선순위: 권장

**행동 설명**:
칸반 보드 상단에 검색바와 필터 옵션이 있다. 검색바에 키워드를 입력하면 제목/설명에 해당 키워드가 포함된 카드만 표시된다. 필터 드롭다운으로 담당자별, 우선순위별 필터링이 가능하다. 활성 필터는 칩(chip) 형태로 표시되며, 칩의 X를 클릭하여 개별 필터를 해제할 수 있다. "필터 초기화" 버튼으로 모든 필터를 한 번에 해제한다.

**UI 와이어프레임 (텍스트)**:
```
┌──────────────────────────────────────────────────┐
│  [🔍 업무 검색...]  [담당자 ▼] [우선순위 ▼]  [초기화] │
│  활성 필터: [김민수 ✕] [높음 ✕]                      │
├──────────────────────────────────────────────────┤
│  (필터링된 칸반 보드)                                │
└──────────────────────────────────────────────────┘
```

**수락 기준**:
- [ ] 검색바에 키워드 입력 시 실시간으로 매칭되는 카드만 표시된다
- [ ] 담당자 드롭다운에서 팀원 선택 시 해당 팀원의 카드만 표시된다
- [ ] 우선순위 드롭다운에서 선택 시 해당 우선순위 카드만 표시된다
- [ ] 복수 필터 동시 적용이 가능하다 (AND 조건)
- [ ] 활성 필터가 칩으로 표시되며, 개별 해제가 가능하다
- [ ] "초기화" 클릭 시 모든 필터가 해제되고 전체 카드가 표시된다

**엣지 케이스**:
- 검색 결과 없음: "검색 결과가 없습니다" 메시지 표시
- 필터 적용 상태에서 카드 드래그앤드롭: 정상 동작

### 3.7 대시보드 — 우선순위: 권장

**행동 설명**:
사이드바에서 "Dashboard"를 클릭하면 업무 현황 통계 화면으로 이동한다. 상단에 핵심 지표 카드(전체 업무 수, 완료율, 진행 중, 마감 임박)가 비대칭 레이아웃으로 배치된다. 중앙에는 상태별 업무 분포 차트(가로 막대 또는 도넛)가 있고, 하단에는 담당자별 업무량 분포와 마감 임박 업무 목록(7일 이내)이 나란히 배치된다.

**UI 와이어프레임 (텍스트)**:
```

│             │                                      │

│ ◉ Dashboard │  │ 전체 업무    │ │완료율 │ │마감임박│   │

│             │  │ (큰 카드)    │ │      │ │      │   │
│             │  └────────────┘ └──────┘ └──────┘   │
│             │                                      │
│             │  ┌────────────────────────────┐      │
│             │  │ 상태별 분포 차트              │     │
│             │  │ ████████░░░░ To Do: 5       │     │
│             │  │ ██████░░░░░ Progress: 3     │     │

│             │  │ ██████████████ Done: 7      │     │

│             │  │ 김민수: ████  │ │ · API 연동 4/12│  │
│             │  │ 이지은: ███   │ │ · UI 수정 4/13 │  │
│             │  │ ...          │ │ · 테스트 4/14  │  │
│             │  └──────────────┘ └──────────────┘  │
└──────────────────────────────────────────────────┘
```

**수락 기준**:
- [ ] 전체 업무 수, 완료율(%), 진행 중 업무 수, 마감 임박 업무 수가 표시된다
- [ ] 통계 수치가 실제 업무 데이터를 정확히 반영한다
- [ ] 상태별 업무 분포 차트가 시각적으로 표시된다
- [ ] 담당자별 업무량 분포가 표시된다
- [ ] 마감 임박(7일 이내) 업무 목록이 마감일 순으로 표시된다
- [ ] 대시보드 수치가 업무 변경(추가/삭제/상태이동) 후 갱신된다

**엣지 케이스**:
- 업무가 0건일 때: 빈 상태 화면 ("아직 업무가 없습니다. 보드에서 첫 업무를 만들어보세요.")
- 마감일 미설정 업무: 마감 임박 목록에서 제외

### 3.8 인앱 알림 — 우선순위: 선택

**행동 설명**:
헤더 영역에 벨 아이콘이 있으며, 읽지 않은 알림이 있으면 뱃지 숫자가 표시된다. 벨 클릭 시 드롭다운 패널이 열리며 최근 알림 목록이 나타난다. 알림 유형: 마감 임박(2일 이내), 새 업무 할당. 각 알림에는 타임스탬프와 관련 업무 정보가 표시되며, 클릭 시 해당 업무 상세 뷰로 이동한다.

**UI 와이어프레임 (텍스트)**:
```
                            ┌──────────────────────┐
                   🔔(3)  → │ 알림                   │
                            │ ──────────────────── │
                            │ 🕐 "API 연동" 마감 2일  │
                            │    전 — 4/12          │
                            │ 📌 "UI 검토" 업무가    │
                            │    회원님에게 할당됨    │
                            │ 🕐 "DB 설계" 마감 임박  │
                            │    — 4/13             │
                            │ ──────────────────── │
                            │ 모두 읽음 처리          │
                            └──────────────────────┘
```

**수락 기준**:
- [ ] 헤더에 벨 아이콘이 표시되며, 읽지 않은 알림 수가 뱃지로 나타난다
- [ ] 벨 클릭 시 알림 드롭다운이 열린다
- [ ] 마감 임박(2일 이내) 업무에 대한 알림이 자동 생성된다
- [ ] 현재 로그인한 사용자에게 할당된 업무의 알림만 표시된다
- [ ] 알림 클릭 시 해당 업무 상세 패널이 열린다
- [ ] "모두 읽음 처리" 클릭 시 뱃지 숫자가 0이 된다

**엣지 케이스**:
- 알림 0건: 벨 아이콘에 뱃지 없음, 드롭다운에 "새 알림이 없습니다" 표시

## 7. 데모 시나리오

데모 발표 시 보여줄 흐름:

1. **시작 — 팀 로그인**: 로그인 화면에서 "김민수" 아바타를 클릭하여 진입. 워크스페이스에 들어온 느낌 전달.
2. **칸반 보드 탐색**: 4개 컬럼에 배치된 업무 카드들을 조망. 카드의 우선순위 색상, 담당자 아바타, 마감일 확인.
3. **드래그앤드롭**: "To Do"에 있는 카드를 "In Progress"로 드래그. 상태가 즉시 변경되는 것을 보여줌.
4. **업무 생성**: "업무 추가" 클릭 → 제목, 설명, 담당자(이지은), 마감일, 우선순위(높음) 입력 → 저장. 새 카드가 보드에 나타남.
5. **업무 상세**: 방금 생성한 카드 클릭 → 슬라이드 패널에서 상세 정보 확인 → 설명 수정 후 저장.
6. **필터링**: 담당자 필터에서 "이지은" 선택 → 이지은의 업무만 표시. 필터 칩 확인 후 초기화.
7. **대시보드**: 사이드바 "Dashboard" 클릭 → 전체 업무 현황 통계 확인. 완료율, 상태별 분포, 마감 임박 업무 조망.
8. **팀 관리**: "Team" 클릭 → 팀원별 업무량 확인 → 특정 팀원 클릭으로 칸반 보드 필터링.
9. **마무리**: 보드로 복귀하여 전체 뷰. "데이터 영속성" — 새로고침해도 모든 데이터 유지됨을 시연.
