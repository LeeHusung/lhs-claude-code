# TaskFlow — 구현 계획

## 기술 스택

| 영역 | 선택 | 선택 이유 |
|------|------|----------|
| Frontend | Next.js (App Router) | 풀스택 단일 프레임워크. SSR로 빠른 초기 로딩. 데모 최단 구현. |
| Backend | Next.js API Routes | 별도 서버 불필요. 프론트와 동일 프로젝트 내 API 구현. |
| Database | SQLite | 사용자-업무 관계형 데이터. 제로 설정, 파일 하나로 영속성 보장, 로컬 데모에 최적. |
| Styling | Tailwind CSS | 유틸리티 기반 빠른 UI 구축. 커스텀 디자인 시스템 쉽게 적용. |

## 디자인 비전

"Focused Workspace" — 깊은 잉크빛 다크 캔버스 위에 따뜻한 앰버/골드 액센트. Linear의 절제된 우아함 + Trello의 직관적 칸반 + 독자적 따뜻한 색감. 보라색 그래디언트, 동일 카드 3열, 제네릭 헤딩 일절 금지. 비대칭 레이아웃과 의도적 색상 팔레트로 "전문가 도구" 인상 전달.

## 태스크 목록

### Phase A: 프로젝트 스캐폴딩 (순차)

#### Task 0: 프로젝트 초기화 + 디자인 시스템 기반
- 설명: Next.js 프로젝트 생성, Tailwind 설정, SQLite 연결, 공통 레이아웃(사이드바 + 헤더 + 메인 영역) 구성. 디자인 비전에 따른 색상 팔레트, 타이포그래피, 기본 테마 설정. 데이터베이스 초기화 및 더미 데이터 시딩.
- 복잡도: 중간
- 의존: 없음
- 참조: spec.md 섹션 2, 2.5, 5, 6
- 병렬 가능: 아니오 (다른 모든 태스크의 기반)

### Phase B: 핵심 기능 구현 (병렬 가능)

#### Task 1: 로그인 화면 — 복잡도: 낮음
- 설명: 팀원 아바타 선택 로그인 + 이메일 폼 토글. 로그인 상태 유지(세션/쿠키). 로그아웃 기능. 비로그인 시 보드 접근 차단.
- 의존: Task 0
- 참조: spec.md 섹션 3.1, 4.1
- 병렬 가능: 예 (Task 2, 3과 병렬)

#### Task 2: 칸반 보드 + 드래그앤드롭 — 복잡도: 높음
- 설명: 4개 컬럼(To Do / In Progress / In Review / Done) 레이아웃. 업무 카드 표시(제목, 우선순위, 담당자 아바타, 마감일). 드래그앤드롭으로 컬럼 간·컬럼 내 이동. 이동 시 서버 저장. 마감 임박 카드 시각 강조.
- 의존: Task 0
- 참조: spec.md 섹션 3.2, 4.2
- 병렬 가능: 예 (Task 1, 3과 병렬)

#### Task 3: 업무 CRUD (생성/편집/삭제) — 복잡도: 중간
- 설명: "업무 추가" 폼 (제목, 설명, 담당자, 마감일, 우선순위). 저장 시 To Do 컬럼에 카드 추가. 기존 업무 편집. 삭제 확인 대화상자 후 삭제. 데이터 영속성 확인.
- 의존: Task 0
- 참조: spec.md 섹션 3.3
- 병렬 가능: 예 (Task 1, 2와 병렬)

### Phase C: 상세·확장 기능 (병렬 가능, Phase B 의존)

- 설명: 카드 클릭 시 오른쪽 슬라이드 패널 열림(애니메이션). 모든 필드 표시 + 인라인 편집. 배경 어둡게 처리. X 또는 바깥 클릭으로 닫기. 편집 후 보드 카드 즉시 반영.

#### Task 5: 팀원 관리 화면 — 복잡도: 낮음

- 의존: Task 0
- 참조: spec.md 섹션 3.5, 4.4
- 병렬 가능: 예 (Task 4, 6, 7과 병렬)

#### Task 6: 필터/검색 기능 — 복잡도: 중간
- 설명: 보드 상단 검색바(키워드 실시간 필터링). 담당자·우선순위 드롭다운 필터. 활성 필터 칩 표시 + 개별 해제. 전체 초기화 버튼. 필터 AND 조합.

- 설명: 핵심 지표 카드 (비대칭 레이아웃). 상태별 분포 차트. 담당자별 업무량. 마감 임박 목록(7일 이내). 실제 데이터 반영.

### Phase D: 선택 기능 + 마무리 (순차)

#### Task 8: 인앱 알림 — 복잡도: 낮음
- 설명: 헤더 벨 아이콘 + 읽지 않은 알림 뱃지. 드롭다운 알림 목록 (마감 임박, 업무 할당). 알림 클릭 시 업무 상세 열기. "모두 읽음 처리". 현재 로그인 사용자 기준 필터.
- 의존: Task 2, Task 4
- 참조: spec.md 섹션 3.8
- 병렬 가능: 아니오

#### Task 9: 통합 + 더미 데이터 검증 + 빌드 확인
- 설명: 전체 화면 간 네비게이션 검증. 더미 데이터(팀원 5명, 업무 17건) 정합성 확인. 데이터 영속성(새로고침) 테스트. 빌드 에러 해결. 콘솔 에러 제거. 데모 시나리오 순서대로 최종 점검.
- 의존: 모든 태스크
- 참조: spec.md 섹션 7 (데모 시나리오)
- 병렬 가능: 아니오

## 병렬 실행 구조 요약

```
Phase A (순차)
  └─ Task 0: 스캐폴딩 + 디자인 시스템

Phase B (병렬 — Task 0 완료 후)
  ├─ Task 1: 로그인
  ├─ Task 2: 칸반 보드 + DnD
  └─ Task 3: 업무 CRUD

Phase C (병렬 — Phase B 완료 후)
  ├─ Task 4: 업무 상세 패널
  ├─ Task 5: 팀원 관리
  ├─ Task 6: 필터/검색
  └─ Task 7: 대시보드

Phase D (순차 — Phase C 완료 후)
  ├─ Task 8: 인앱 알림 (선택)
  └─ Task 9: 통합 + 최종 검증
```

## 리스크 및 주의사항

- **드래그앤드롭 복잡도**: Task 2가 가장 기술적으로 복잡. 라이브러리 선택과 서버 동기화가 핵심.
- **데모 일정**: 2026-04-15 (5일 후). 총 10개 태스크, 병렬 3단계 구조로 충분히 소화 가능.
- **선택 기능(Task 8)**: 시간이 부족하면 스킵 가능. 핵심 기능에 영향 없음.
