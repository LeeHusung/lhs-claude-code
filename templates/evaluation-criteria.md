# 평가 기준

Evaluator Agent가 생성된 앱을 검증할 때 사용하는 채점 기준.
Playwright MCP 또는 Claude Preview MCP로 실제 앱을 조작하며 평가한다.

GAN 영감의 적대적 평가: Evaluator는 Generator의 작업을 **회의적으로** 검증한다.
합법적인 문제를 발견했으면 중요하지 않다고 스스로 설득하지 않는다.
표면적 테스트에 그치지 않고 엣지 케이스를 적극 탐사한다.

---

## 4차원 평가 체계 (100점)

원문 참조: Anthropic "Harness design for long-running application development"

Claude는 기술적으로 깨끗한 코드를 쓰지만, 디자인은 "무해하지만 따분한" 결과를 생성하는 경향이 있다.
이를 극복하기 위해 디자인 품질과 독창성에 더 큰 가중치를 부여한다.

### 1. 기능성 — Functionality (30점)

프론트엔드와 백엔드를 포함한 **전체 스택**이 사용자 시나리오대로 동작하는가?

| 항목 | 배점 | 설명 |
|------|------|------|
| 핵심 기능 동작 | 10점 | spec.md의 수락 기준대로 모든 필수 기능이 동작하는가 (UI 조작 → API 호출 → 결과 표시) |
| 데이터 영속성 | 5점 | 데이터가 실제로 저장되는가? 새로고침 후에도 유지되는가? display-only가 아닌가? |
| API/서버 정상 동작 | 5점 | API가 올바른 응답을 반환하는가? 서버 에러(500, 422) 없는가? |
| 에러 없음 | 5점 | 브라우저 콘솔 에러(JS, 네트워크, 404) + 서버 로그 에러 없는가 |
| 네비게이션/플로우 | 5점 | 화면 간 이동이 직관적이고, 사용자가 길을 잃지 않는가 |

**우선순위별 감점**:
- 필수 기능 미동작: 해당 점수 전액 + 추가 5점 감점
- 권장 기능 미동작: 해당 점수 전액 감점
- 선택 기능 미구현: 감점 없음

**display-only 경계**: UI에 버튼/폼이 있지만 실제 백엔드 연동이 안 되는 경우, 해당 기능은 "미동작"으로 판정한다.

### 2. 디자인 품질 — Design Quality (25점)

개별 부분이 아닌 **응집된 전체**로 느껴지는가?

| 항목 | 배점 | 설명 |
|------|------|------|
| 시각적 응집력 | 10점 | 색상, 타이포그래피, 레이아웃이 하나의 통일된 시스템으로 느껴지는가 |
| 분위기/정체성 | 10점 | 앱이 독특한 분위기나 브랜드 정체성을 갖고 있는가 (단순 기본 UI 아닌) |
| 레이아웃 완성도 | 5점 | 오버랩, 오버플로우, 정렬 불량 없이 깔끔한가 |

**핵심 질문**: "이 앱이 전문적으로 디자인된 것처럼 보이는가, 아니면 프레임워크 기본값으로 보이는가?"

### 3. 독창성 — Originality (25점)

맞춤형 디자인 결정의 증거가 있는가, 아니면 템플릿과 기본값인가?

| 항목 | 배점 | 설명 |
|------|------|------|
| 커스텀 디자인 결정 | 10점 | 색상 팔레트, 레이아웃, 인터랙션에서 의도적 선택이 보이는가 |
| AI 패턴 탈피 | 10점 | 보라색 그래디언트, 둥근 카드 나열, 제네릭 히어로 섹션 등 전형적 AI 생성 패턴을 피했는가 |
| 창의적 시도 | 5점 | 예상치 못한 레이아웃, 인터랙션, 시각적 접근이 있는가 |

**핵심 질문**: "이 앱이 다른 AI 생성 앱들과 구별되는가?"

**감점 트리거** (각 -3점):
- 보라색/파란색 그래디언트 히어로 섹션
- 동일 크기의 둥근 카드 3열 나열
- "Welcome to [앱이름]" 제네릭 헤딩
- stock photo 느낌의 일러스트/아이콘 남용

### 4. 기술적 완성도 — Craft (20점)

프론트엔드와 백엔드 모두의 기술적 실행 품질.

**프론트엔드 완성도 (10점)**:

| 항목 | 배점 | 설명 |
|------|------|------|
| 타이포그래피/간격 | 4점 | 명확한 계층, padding/margin 균일 |
| 반응형 | 3점 | 375px/768px에서 레이아웃이 적응하는가 (요구 시) |
| 전환/로딩 | 3점 | 화면 이동 시 끊김 없고, 로딩 인디케이터가 있는가 |

**백엔드 완성도 (10점)**:

| 항목 | 배점 | 설명 |
|------|------|------|
| API 설계 일관성 | 4점 | 응답 형식 통일, 적절한 HTTP 상태코드, RESTful 규약 준수 |
| 데이터 무결성 | 3점 | CRUD가 정상 동작하는가, 관계 데이터가 깨지지 않는가 |
| 에러 처리 | 3점 | 잘못된 요청에 적절한 에러 응답 (500 대신 400/404), 서버 크래시 없음 |

## 합격 기준

| 조건 | 결과 | 다음 행동 |
|------|------|----------|
| 총점 ≥ 70점 | **PASS** | Deliver 단계로 진행. 딜리버리 요약 생성. |
| 총점 < 70점, 라운드 < 3 | **FAIL** | 수정 지시를 claude-progress.txt에 기록. 같은 세션에서 수정 후 재검증. |
| 총점 < 70점, 라운드 = 3 | **HALT** | 사용자에게 에스컬레이션. 수동 개입 필요. |

## 수정 지시 작성 가이드 (FAIL 시)

FAIL 시 Evaluator는 다음 형식으로 수정 지시를 작성한다.
Generator(Sonnet)가 모호함 없이 수정할 수 있을 만큼 구체적이어야 한다.

```markdown
## 수정 지시 (라운드 {N})

### 수정 1
- **파일**: {정확한 파일 경로}
- **문제**: {무엇이 잘못되었는지 구체적으로}
- **필요한 수정**: {어떻게 고쳐야 하는지}
- **검증 방법**: {수정 후 어떻게 확인하는지}

### 수정 2
...
```

**규칙**:
- 아키텍처 변경 제안 금지. 기존 구조 내에서 타겟 수정만.
- 각 수정 지시는 하나의 구체적 문제에 대응.
- "전반적으로 개선" 같은 모호한 지시 금지.

## Few-shot 평가 예시

### 예시 1: PASS (82점)

**기능성 (25/30)**:
- 핵심 기능: 9/10 — 로그인, 대시보드, 검색 모두 정상. 특수문자 입력 시 빈 결과 표시 (크래시 아님).
- 데이터 영속성: 4/5 — 대시보드 데이터 새로고침 후 유지. 검색 필터 상태 미보존 (-1).
- API/서버: 5/5 — 모든 API 200/201 정상 응답. 서버 에러 0건.
- 에러 없음: 5/5 — 콘솔 에러 0건, 네트워크 에러 0건.
- 네비게이션: 4/5 — 뒤로가기 시 이전 필터 상태 미보존 (-1).

**디자인 품질 (21/25)**:
- 시각적 응집력: 8/10 — 다크 테마가 일관되나, 모달 배경색이 약간 불일치.
- 분위기/정체성: 8/10 — "전문 분석 도구" 느낌이 확실함. 기본 UI 이상.
- 레이아웃: 5/5 — 깨짐 없음.

**독창성 (18/25)**:
- 커스텀 결정: 8/10 — 색상 팔레트가 의도적. 사이드바 네비게이션에 마이크로 인터랙션.
- AI 패턴 탈피: 6/10 — 히어로 섹션 없음 (좋음). 그러나 카드 레이아웃이 다소 제네릭 (-4).
- 창의적 시도: 4/5 — 대시보드 차트에 커스텀 애니메이션.

**기술적 완성도 (18/20)**:
- 프론트엔드 (9/10): 타이포 계층 명확. 간격 대부분 일관 — 모바일 카드 간격 불균일(-1). 반응형 375px 양호.
- 백엔드 (9/10): API 응답 형식 일관. CRUD 정상. 빈 대시보드 초기 상태 미처리(-1).

### 예시 2: FAIL (48점)

**기능성 (12/30)**:
- 핵심 기능: 2/10 — 로그인 정상. 결제 버튼 클릭 시 TypeError (필수 미동작 → 추가 -5). 검색 버튼 존재하나 핸들러 미연결.
- 데이터 영속성: 1/5 — 결제 데이터 저장 불가. 검색 자체가 미동작.
- API/서버: 1/5 — POST /api/checkout 500 에러. GET /api/search 404.
- 에러 없음: 0/5 — JS TypeError 1건, 서버 500 1건, 404 리소스 2건.
- 네비게이션: 3/5 — 기본 이동 가능하나 일부 링크 미연결.

수정 지시:
### 수정 1
- **파일**: src/components/Checkout.tsx
- **문제**: 결제 함수에서 amount가 null일 때 TypeError
- **필요한 수정**: amount null 체크 추가
- **검증 방법**: 결제 버튼 클릭 시 에러 없이 플로우 완료

### 수정 2
- **파일**: src/pages/Search.tsx + server/routes/search.ts
- **문제**: 프론트 onSubmit 미바인딩 + 백엔드 /api/search 라우트 미등록
- **필요한 수정**: onSubmit={handleSearch} 추가 + search 라우트 등록
- **검증 방법**: 검색어 입력 후 API 200 응답 + 결과 표시

### 수정 3
- **파일**: public/
- **문제**: favicon.ico, logo.png 404
- **필요한 수정**: 파일 추가 또는 참조 제거
- **검증 방법**: 콘솔/네트워크 404 에러 없음

**디자인 품질 (10/25)**:
- 시각적 응집력: 4/10 — 폰트 2종 혼용, 버튼 스타일 불일치.
- 분위기/정체성: 3/10 — 기본 프레임워크 UI 그대로. 정체성 없음.
- 레이아웃: 3/5 — 메인 페이지 사이드바+컨텐츠 오버랩.

**독창성 (8/25)**:
- 커스텀 결정: 3/10 — 기본 색상, 기본 레이아웃. 의도적 선택 없음.
- AI 패턴 탈피: 2/10 — 보라색 그래디언트 히어로(-3), 동일 카드 3열(-3), 제네릭 헤딩(-3).
- 창의적 시도: 3/5 — 특별한 시도 없음.

**기술적 완성도 (8/20)**:
- 프론트엔드 (5/10): 타이포 계층 불명확(-2). 간격 불균일(-1). 모바일 네비게이션 깨짐(-2).
- 백엔드 (3/10): /api/checkout이 500 반환(-3). /api/search 라우트 미존재(-2). 에러 응답에 메시지 없음(-2).
